
# Logseq Text Toolkit v1.1.3 发布说明

## 🐛 修复
- **修复 Markdown 格式转换为 hiccup 时错误添加引号的问题**
  - 解决了 `**扫描越久 → STW（暂停时间）越长 → 服务卡顿**` 被错误转换为 `[:b "扫描越久 → ..."]` 的问题
  - 现在正确转换为 `[:b 扫描越久 → STW（暂停时间）越长 → 服务卡顿]`（不添加引号）

## 📝 详细改进

### 核心修复规则
在 `parseNestedFormat` 函数中更新了引号处理逻辑：
- 从 Markdown 格式（如 `**粗体**`、`*斜体*` 等）转换为 hiccup 时，内容不添加引号
- 保持已有的 hiccup 格式不变（如 `[:b "带空格的文本"]` 中的引号保持原样）
- 如果内容包含 hiccup 片段，也不添加引号

### 修改的函数
1. `parseNestedFormat` - 重新实现了 Markdown 格式到 hiccup 的转换逻辑
2. `wrapWithQuotesIfNeeded` - 增加检测文本包含 hiccup 片段的逻辑，避免错误包裹引号
3. `handleNestedQuotes` - 增加 `nestedContainsHiccup` 检测

### 修复示例
**修复前：**
```
输入: "数组越大 → **扫描越久 → STW（暂停时间）越长 → 服务卡顿**。"
输出: "[:span.blue \"数组越大 → [:b \"扫描越久 → STW（暂停时间）越长 → 服务卡顿\"]。\"]"
```

**修复后：**
```
输入: "数组越大 → **扫描越久 → STW（暂停时间）越长 → 服务卡顿**。"
输出: "[:span.blue 数组越大 → [:b 扫描越久 → STW（暂停时间）越长 → 服务卡顿]。]"
```
