# 产品优化文档

## 优化点 1：CSS 文件打包优化

### 功能描述
支持将 CSS 文件打包到插件根目录，使用户可以通过修改 CSS 文件来自定义样式，并且修改后通过重新加载插件即可生效。

### 技术实现方案
1. **构建配置修改**：
   - 修改 Vite 构建配置，将 CSS 文件输出到插件根目录
   - 确保 CSS 文件在构建过程中被正确处理

2. **样式加载方式**：
   - 插件启动时动态加载根目录的 CSS 文件
   - 支持热重载，当 CSS 文件变化时自动重新加载

3. **用户使用流程**：
   - 用户安装插件后，在插件目录中找到 CSS 文件
   - 用户修改 CSS 文件来自定义样式
   - 用户在 Logseq 中重新加载插件
   - 插件重新加载后，新的样式立即生效

### 优势
- **灵活性**：用户可以根据自己的喜好自定义插件样式
- **实时反馈**：修改后重新加载即可看到效果，无需重新构建插件
- **低门槛**：用户不需要了解插件开发，只需修改 CSS 文件即可

## 优化点 2：国际化配置扩展

### 功能描述
在配置中增加基础数据表，特别是语言表，包含语言代码和关联的 JSON 文件。设置对语言的下拉数据来源为该表，支持用户自定义设置自己的国际化展示。同时，其他配置字段也可以采用类似的方式进行扩展。

### 技术实现方案
1. **数据表结构**：
   - **语言表**：包含语言代码、语言名称、关联的 JSON 文件路径等字段
   - **其他数据表**：根据需要扩展其他配置相关的数据表

2. **配置界面**：
   - 语言选择下拉框的数据来源为语言表
   - 支持用户添加、编辑、删除语言条目
   - 支持用户上传或编辑语言对应的 JSON 文件

3. **国际化加载**：
   - 插件启动时根据用户选择的语言加载对应的 JSON 文件
   - 支持动态切换语言，无需重启插件

4. **扩展性**：
   - 其他配置字段也可以采用类似的数据表方式进行管理
   - 提供统一的配置管理界面，方便用户进行配置

### 优势
- **可扩展性**：用户可以添加自己的语言支持，无需修改插件代码
- **灵活性**：用户可以自定义语言的翻译内容
- **一致性**：通过统一的数据表管理，确保配置的一致性和可维护性
- **用户友好**：提供直观的配置界面，降低用户使用门槛
