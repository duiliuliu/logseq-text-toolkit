## 1. 文本格式化

### 1.1 基本操作

文本格式化功能会在你选中文本时自动弹出工具栏，支持以下操作：

| 功能 | 说明 | 使用方法 |
|------|------|---------|
| 加粗 | 将文本设为粗体 | 选中文本 → 点击 ** |
| 斜体 | 将文本设为斜体 | 选中文本 → 点击 * |
| 删除线 | 添加删除线效果 | 选中文本 → 点击 ~~ |
| 高亮 | 背景高亮显示 | 选中文本 → 选择颜色按钮 |
| 文本颜色 | 改变文字颜色 | 选中文本 → 选择颜色按钮 |
| 下划线 | 添加彩色下划线 | 选中文本 → 选择颜色按钮 |
| 代码 | 行内代码格式化 | 选中文本 → 点击 ` |
| 文件链接 | 快速插入文件链接 | 点击📎按钮 |

### 1.2 工具栏使用技巧

- **快捷键支持**：工具栏支持快捷键配置，可在设置中自定义
- **按钮国际化**：工具栏按钮使用 emoji 图标，支持多语言显示
- **自动弹出**：选中文本后工具栏会自动在光标位置弹出

### 1.3 颜色选项

高亮、文本颜色、彩色下划线支持以下颜色：

- 🔴 红色
- 🟡 黄色
- 🔵 蓝色
- 🟢 绿色
- 🟣 紫色

## 2. 任务进度追踪

### 2.1 基本使用

在任意块中使用任务进度渲染器：

```markdown
- 我的项目 {{renderer :taskprogress}}
  - 任务1 #task status:: todo
  - 任务2 #task status:: doing
  - 任务3 #task status:: done
```

### 2.2 支持的任务状态

| 状态 | 标识 | 说明 |
|------|------|------|
| todo | 待办 | 尚未开始的任务 |
| doing | 进行中 | 正在执行的任务 |
| in-review | 审核中 | 等待审核的任务 |
| done | 已完成 | 已完成的任务 |
| waiting | 等待中 | 等待其他任务的任务 |
| canceled | 已取消 | 被取消的任务 |

### 2.3 展示样式

任务进度支持多种展示样式：

| 样式 | 说明 | 命令参数 |
|------|------|---------|
| 微型圆环 | Mini Circle | `mini-circle` |
| 点阵进度 | Dot Matrix | `dot-matrix` |
| 状态光标 | Status Cursor | `status-cursor` |
| 进度胶囊 | Progress Capsule | `progress-capsule` |
| 阶梯进度 | Step Progress | `step-progress` |

**指定样式示例**：
```markdown
{{renderer :taskprogress, step-progress}}
{{renderer :taskprogress, display=step-progress, size=small}}
```

### 2.4 嵌套统计

任务进度支持嵌套层级的统计：

| 嵌套层级 | 说明 |
|---------|------|
| 仅当前层 | 只统计直接子任务 |
| 向下2层 | 统计当前层和下一层 |
| 向下3层 | 统计当前层和下两层 |
| 全部层级 | 统计所有嵌套层级 |

**进阶设置**：
- **仅统计叶子任务**：开启后只统计没有子任务的任务
- **显示嵌套指示器**：显示任务嵌套深度

### 2.5 标签格式

任务进度支持多种标签格式：

| 格式 | 示例 | 说明 |
|------|------|------|
| 分数 | 10/20 | 显示完成数/总数 |
| 百分比 | 50% | 显示完成百分比 |
| 两者 | 10/20 50% | 同时显示分数和百分比 |

### 2.6 成就烟花

开启"完成所有任务时绽放烟花"选项，当所有任务完成时会有绚丽的烟花效果。

### 2.7 自定义颜色

在设置中可以自定义各状态的显示颜色：

```markdown
状态颜色设置：
- 待办：灰色系
- 进行中：蓝色系
- 已完成：绿色系
- 审核中：黄色系
- 等待中：橙色系
- 已取消：红色系
```

## 3. Heatmap 热力图

### 3.1 基本使用

```markdown
{{renderer :heatmap, month, tag=work}}
```

### 3.2 视图类型

| 视图 | 参数 | 说明 |
|------|------|------|
| 年度视图 | `year` | 显示整年的热力图 |
| 月度视图 | `month` | 显示单月热力图 |
| 周度视图 | `week` | 显示单周热力图 |

### 3.3 查询方式

| 查询类型 | 参数格式 | 示例 |
|---------|---------|------|
| 标签查询 | `tag=标签名` | `tag=work` |
| 页面查询 | `page=页面名` | `page=My Page` |
| 属性查询 | `property=属性::值` | `property=category::work` |

### 3.4 参数组合示例

```markdown
{{renderer :heatmap, view=year, tag=work}}
{{renderer :heatmap, view=month, page=My Page}}
{{renderer :heatmap, view=week, property=category::work, week=20, year=2026}}
```

### 3.5 配色方案

Heatmap 支持自定义颜色方案：

- **最小颜色**：数据为0时显示的颜色
- **最大颜色**：数据最多时显示的颜色
- **颜色公式**：
  - 简化模式：线性渐变
  - 加权模式：非线性权重计算

### 3.6 页面自动创建（开发中）

可配置自动创建月度/周度总结页面：

- **页面名称模板**：自定义生成的页面名称格式
- **Logseq 模板**：选择用于生成内容的模板

## 4. 块视图 (Block View)

### 4.1 简介

块视图是一个强大的功能，允许你以多种视图方式展示 Logseq 中的块内容。

**支持视图类型**：
- 📋 **列表视图 (List)**：传统列表展示
- 📊 **表格视图 (Table)**：表格形式展示
- 🖼️ **画廊视图 (Gallery)**：卡片画廊展示
- 📋 **看板视图 (Board)**：看板列展示

### 4.2 基本使用

```markdown
{{renderer :block-view}}
```

指定视图类型：
```markdown
{{renderer :block-view, view=table}}
{{renderer :block-view, view=gallery}}
{{renderer :block-view, view=board}}
```

### 4.3 主题支持

Block View 支持多种预设主题：

| 主题 | 说明 | 特点 |
|------|------|------|
| Default | 默认主题 | 简洁清晰 |
| Notion | Notion风格 | 类似Notion的简约风格 |
| Linear | Linear风格 | 科技感强的深色系 |
| Dark | 深色主题 | 全方位深色配色 |
| Gradient | 渐变主题 | 渐变色设计 |
| Tana | Tana风格 | 类似Tana的柔和配色 |
| Custom | 自定义主题 | 用户自定义颜色配置 |

**指定主题示例**：
```markdown
{{renderer :block-view, theme=notion}}
{{renderer :block-view, view=table, theme=linear}}
```

### 4.4 视图切换栏

可以显示或隐藏视图切换栏：

```markdown
{{renderer :block-view, hideBar=true}}
```

- `hideBar=true`：隐藏切换栏，使用默认视图
- `hideBar=false`：显示切换栏（默认）

### 4.5 自定义主题配置

在设置 → Block View 中可以配置自定义主题：

#### 表格视图 (Table)

| 配置项 | 说明 |
|--------|------|
| 边框颜色 | 表格边框颜色 |
| 表头背景色 | 表头单元格的背景颜色 |
| 表头文字色 | 表头文字的颜色 |
| 单元格文字色 | 表格内容文字的颜色 |
| 表头边框色 | 表头与其他行的分隔线颜色 |
| 表头高度 | 表头行的高度 |
| 行背景色 | 数据行背景色 |
| 行悬停背景色 | 鼠标悬停时行背景色 |
| 行边框色 | 行之间的分隔线颜色 |
| 单元格内边距 | 单元格内文字与边框的距离 |
| 表格圆角 | 表格四个角的圆角大小 |

#### 画廊视图 (Gallery)

| 配置项 | 说明 |
|--------|------|
| 边框颜色 | 卡片边框颜色 |
| 卡片背景色 | 卡片背景颜色 |
| 卡片悬停背景色 | 鼠标悬停时卡片背景色 |
| 表头边框色 | 标题区边框颜色 |
| 表头背景色 | 标题区背景颜色 |
| 表头文字色 | 标题文字颜色 |
| 卡片文字色 | 卡片内容文字颜色 |
| 卡片圆角 | 卡片四个角的圆角大小 |
| 卡片阴影 | 卡片阴影效果 |

#### 看板视图 (Board)

| 配置项 | 说明 |
|--------|------|
| 边框颜色 | 列边框颜色 |
| 列背景色 | 列背景颜色 |
| 列悬停背景色 | 鼠标悬停时列背景色 |
| 表头背景色 | 列标题背景色 |
| 表头文字色 | 列标题文字色 |
| 卡片背景色 | 卡片背景色 |
| 卡片文字色 | 卡片内容文字色 |
| 卡片边框色 | 卡片边框色 |
| 卡片圆角 | 卡片圆角大小 |

### 4.6 参数速查表

| 参数 | 说明 | 可选值 |
|------|------|--------|

## 6. 工具栏配置

### 6.1 基本配置

工具栏按钮可以通过 JSON 格式配置：

```json
[
  {
    "id": "format-bold",
    "label": "加粗",
    "icon": "**",
    "invoke": "editor/insert-batch-edit",
    "invokeParams": {
      "texts": [["**{{selected}}**", "{{selected}}**"]]
    }
  }
]
```

### 6.2 配置字段说明

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| id | string | 是 | 唯一标识符 |
| label | string | 是 | 按钮显示名称 |
| icon | string | 是 | 按钮图标（支持 emoji） |
| invoke | string | 是 | 调用的 Logseq 命令 |
| invokeParams | object | 是 | 命令参数 |
| hidden | boolean | 否 | 是否隐藏按钮 |
| binding | string | 否 | 快捷键绑定（开发中） |
| subItems | array | 否 | 分组子元素 |

### 6.3 常用命令

| 命令 | 说明 | 用途 |
|------|------|------|
| `editor/insert-batch-edit` | 批量编辑插入 | 格式化文本 |
| `editor/insert-space` | 插入空格 | 调整格式 |
| `page/insert-file-link` | 插入文件链接 | 添加文件引用 |

### 6.4 重置工具栏

如果配置出错，可以使用"重置为默认值"选项恢复默认配置。

## 7. 主题与多语言

### 7.1 主题设置

插件支持以下主题设置选项：

| 选项 | 说明 |
|------|------|
| 跟随系统 | 自动跟随 Logseq 的主题设置 |
| 浅色模式 | 强制使用浅色主题 |

### 7.2 语言设置

支持以下语言：

| 语言 | 选项名称 |
|------|---------|
| 中文 | 简体中文 |
| English | English |
| 日本語 | 日本語 |

可以选择"跟随系统"自动匹配 Logseq 的语言设置。

### 7.3 CSS 自定义

可以通过 Logseq 的 CSS 设置覆盖插件的默认样式：

```css
/* 自定义工具栏样式 */
.ltt-toolbar {
  --ltt-bg: #ffffff;
  --ltt-border: #e5e7eb;
}

/* 自定义任务进度颜色 */
.ltt-task-progress {
  --ltt-done-color: #22c55e;
}
```

## 8. 常见问题

### Q1: 工具栏没有弹出？
**A**: 请确保选中了文本内容，工具栏只会在选中文本时自动弹出。

### Q2: 任务进度不显示？
**A**: 检查任务是否正确使用了 `#task` 标签和 `status::` 属性。

### Q3: Block View 主题不生效？
**A**: 确保在设置中选择了正确的视图类型和主题，然后刷新页面。

### Q4: 如何重置所有设置？
**A**: 在设置面板的各个 Tab 中都有"重置为默认值"按钮，可以单独重置某个功能板块的设置。

### Q5: 支持哪些 Logseq 版本？
**A**: 建议使用最新版本的 Logseq 以获得最佳兼容性和最新功能。
