# 操作规则
## 一、输出前置必做事项
1. **需求闭环校验**
输出前必须逐项核对，确保100%完成用户全部要求，无遗漏、无偏差、无额外添加。

2. **会话总结**
固定输出三要素：
- 用户需求描述
- 解决方案
- 最终结果/交付物

3. **更新日志（强制记录）**
按以下分类记录：
- **时间**：YYYY-MM-DD HH:MM:SS
- **版本**：系统代码版本号或commit记录
- **用户描述**：精简后的用户需求描述（代码等不用贴）
- **更新内容**：
  - 新增：新增功能、规则、流程、接口
  - 更新：逻辑优化、表述优化、代码优化
  - 删除：清理无用代码、废弃功能、冗余规则
  - 问题修复（必须完整三要素）：
    - 问题描述
    - 问题原因
    - 解决方案

4. **错题本（强制记录）**
满足以下任一条件必须记录：
- 用户明确说明：这是 **AI 犯错 case**
- 用户反馈：**这个问题之前已经出现过**

5. **优化待办项**
与本次目的无直接达成/解决关系，但是对整个产品有增益的事项：
- **分类**：产品UI、前端代码、后端代码、技术框架、性能优化、安全优化、其他
- **详细描述**：详细说明优化内容，可添加图片
- **优先级**：高、中、低
- **优先级原因**：说明为什么设定此优先级

---

## 二、固定输出要求
1. 必须输出：**当前启动的测试服务器地址**
2. 结构清晰、关键信息醒目
3. 内容精简、不冗余、不跑题

---

## 三、编程开发强制规则（核心）
1. **接口先行，实现后补**
先定义接口、入参、出参、结构，再补全具体实现逻辑。

2. **代码极简**
不添加任何不需要的功能、变量、依赖；
不用的代码必须立即清理删除。

3. **模块化编写（Modularize）**
- 功能拆分独立模块
- **禁止随意修改现有核心逻辑**
- 用模块化扩展防止系统崩溃

4. **测试驱动输出（Test-Driven）**
- 需求指定测试用例时，**先生成测试代码，再生成功能代码**
- 优先支持 pytest 格式

5. **增量式修改（Incremental）**
- 一次只做**一个修改点**
- 修改完成后**立即测试验证**
- 确保稳定后再进行下一次修改
