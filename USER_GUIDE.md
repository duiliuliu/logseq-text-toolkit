
# Logseq Text Toolkit 插件用户操作文档

## 概述

`logseq-text-toolkit` 是一个为 Logseq 提供自定义文本包裹、替换、格式化与快捷键功能的插件。本文档将详细介绍如何安装、配置和使用该插件。

## 1. 插件安装

### 方式一：通过 Logseq 插件市场安装（推荐）
1. 打开 Logseq
2. 点击右上角菜单 → **插件**
3. 在插件市场中搜索 `logseq-text-toolkit`
4. 点击 **安装**
5. 安装完成后，插件会自动启用

### 方式二：通过开发者模式手动安装
（详见 [4. 开发者模式设置](#4-开发者模式设置) 章节）

## 2. 功能操作说明

### 2.1 文本格式化功能

#### 2.1.1 基本样式
1. **操作步骤**：
   - 在 Logseq 中选择任意文本块
   - 进入编辑模式
   - 选中需要格式化的文本
   - 此时会弹出格式化工具栏
   - 点击工具栏上的相应按钮

2. **功能效果**：
   - **加粗**：将选中的文本包裹在 `**` 之间，如 `**文本**`
   - **斜体**：将选中的文本包裹在 `*` 之间，如 `*文本*`
   - **删除线**：将选中的文本包裹在 `~~` 之间，如 `~~文本~~`

3. **效果图**：
   ![基本样式效果](screenshots/basic_styles.png)

#### 2.1.2 高亮功能
1. **操作步骤**：
   - 选中需要高亮的文本
   - 点击工具栏中的高亮按钮
   - 选择相应的颜色

2. **功能效果**：
   - **黄色高亮**：将选中的文本包裹在 `==` 之间，如 `==文本==`
   - **红色高亮**：将选中的文本包裹在 `[[#red]]==` 和 `==` 之间，如 `[[#red]]==文本==`
   - **绿色高亮**：将选中的文本包裹在 `[[#green]]==` 和 `==` 之间，如 `[[#green]]==文本==`
   - **蓝色高亮**：将选中的文本包裹在 `[[#blue]]==` 和 `==` 之间，如 `[[#blue]]==文本==`

3. **效果图**：
   ![高亮效果](screenshots/highlight_effects.png)

#### 2.1.3 文本颜色

### 2.4 使用快捷键
插件支持自定义快捷键。默认配置中已包含一些快捷操作，您也可以在设置中自定义。

**默认快捷键**：
- **移除格式**：`mod+shift+x`

**自定义快捷键**：
1. 在插件设置中找到相应的命令
2. 在 `binding` 字段中设置您想要的快捷键组合
3. 保存配置并重启插件

### 2.5 工具栏样式自定义
您可以通过添加自定义 CSS 来修改工具栏的样式。例如：

```css
/* 工具栏背景颜色 */
:root {
  --kef-wrap-tb-bg: #333e;
}
:root.dark {
  --kef-wrap-tb-bg: #777e;
}

/* 工具栏样式 */
#kef-wrap-toolbar {
  background: #333;
}

/* 工具栏按钮样式 */
.kef-wrap-tb-item {
}

/* 工具栏按钮悬停样式 */
.kef-wrap-tb-item:hover {
  filter: drop-shadow(0 0 3px #fff);
}

/* 图标样式 */
.kef-wrap-tb-item img {
  width: 20px;
  height: 20px;
}
```

## 3. 配置详解

### 3.1 打开插件配置
1. 在 Logseq 中，点击右上角菜单 → **插件**
2. 找到 `logseq-text-toolkit`，点击右侧的 **设置** 按钮
3. 在打开的配置界面中，您可以编辑和自定义插件的行为

### 3.2 主要配置项说明

#### 基础配置
```json
{
  "disabled": false,          // 是否禁用插件
  "toolbar": true,            // 是否显示工具栏
  "toolbarShortcut": ""       // 工具栏显示/隐藏的快捷键
}
```

#### 包裹命令配置 (wrap-*)
包裹命令用于将选中的文本用特定的格式包裹起来。

**配置结构：**
```json
{
  "wrap-bold": {
    "label": "Bold",                // 显示在工具栏的标签
    "binding": "",                  // 快捷键（可选）
    "template": "**$^**",          // 包裹模板，$^ 代表选中的文本
    "icon": "&lt;svg ...&gt;&lt;/svg&gt;"    // 图标（SVG 格式）
  }
}
```

**模板占位符说明：**
- `$^`：表示选中的文本
- `$%`：表示插件命令的返回值（配合 `pluginCommand` 使用）

#### 替换命令配置 (repl-*)
替换命令用于通过正则表达式替换选中的文本。

**配置结构：**
```json
{
  "repl-clear": {
    "label": "Remove formatting",    // 显示在工具栏的标签
    "binding": "mod+shift+x",        // 快捷键
    "regex": "...",                  // 匹配的正则表达式
    "replacement": "$1$2..."         // 替换模板
  }
}
```

#### 分组配置 (group-*)
您可以将多个命令分组，它们会在工具栏中一起显示。

**配置结构：**
```json
{
  "group-style": {
    "wrap-bold": { ... },
    "wrap-italic": { ... }
  }
}
```

### 3.3 内置格式化类型
插件内置了多种常用的格式化类型：

| 类别 | 功能 | 模板示例 |
|------|------|----------|
| 引用 | 页面引用、链接 | `[[$^]]`, `[$^]()` |
| 样式 | 加粗、斜体、删除线 | `**$^**`, `*$^*`, `~~$^~~` |
| 高亮 | 背景高亮（多种颜色） | `[:mark.red $^]`, `==$^==` |
| 文本颜色 | 字体颜色（多种颜色） | `[:span.red $^]` |
| 下划线 | 下划线高亮 | `[:mark.red-underline $^]` |
| 其他 | 内联代码、公式 | `` `$^` ``, `$$^$` |

## 4. 开发者模式设置

如果您需要对插件进行开发、调试或修改，或者想要安装未发布到插件市场的版本，可以使用开发者模式。

### 4.1 开启 Logseq 开发者模式
1. 打开 Logseq
2. 点击右上角菜单 → **设置**
3. 在 **高级** 选项卡中，找到 **开发者模式**
4. 勾选 **开启开发者模式**

### 4.2 准备插件代码
1. 将插件代码下载或克隆到本地目录
2. 确保目录结构包含以下文件：
   - `package.json`（插件配置文件）
   - `dist/index.html`（编译后的插件入口文件）
   - 其他源文件

### 4.3 构建插件
如果您修改了源代码，需要重新构建：

1. 确保已安装 Node.js 和 yarn
2. 在插件目录中打开终端
3. 运行以下命令安装依赖：
   ```bash
   yarn install
   ```
4. 运行构建命令：
   ```bash
   yarn build
   ```
5. 构建完成后，会在 `dist` 目录中生成编译后的文件

### 4.4 在 Logseq 中加载插件
1. 在 Logseq 中，点击右上角菜单 → **插件**
2. 点击 **加载未打包的插件** 按钮
3. 在弹出的文件选择器中，选择插件的根目录（包含 `package.json` 的目录）
4. 插件加载成功后会自动启用

### 4.5 调试插件
1. 在 Logseq 中按 `Ctrl+Shift+I`（或 `Cmd+Option+I`）打开开发者工具
2. 在 Console 标签中可以查看插件的日志输出
3. 在 Sources 标签中可以设置断点进行调试

## 5. WebURL 插件设置

### 5.1 什么是 WebURL 插件
Logseq 支持通过 URL（web URL）加载插件，这种方式适用于：
- 快速测试插件
- 分享插件给他人测试
- 在不安装的情况下体验插件

### 5.2 使用 WebURL 加载插件
1. 将插件托管到可以通过 HTTP/HTTPS 访问的位置（如 GitHub Pages、Vercel、Netlify 等）
2. 确保 `dist` 目录可以公开访问
3. 在 Logseq 中，点击右上角菜单 → **插件**
4. 点击 **从 URL 加载插件**
5. 输入插件的 URL（指向 `dist/index.html`）
6. 点击 **加载**

### 注意事项
- 使用 WebURL 加载的插件在 Logseq 关闭后不会保留
- 某些浏览器安全策略可能会限制 WebURL 插件的功能
- 建议仅用于测试目的，日常使用请安装正式版本

### 路径一：普通用户安装和使用
```
1. 打开 Logseq
   ↓
2. 进入插件市场
   ↓
3. 搜索并安装 logseq-text-toolkit

```
1. 打开 Logseq
   ↓
2. 进入插件设置
   ↓
3. 找到 logseq-text-toolkit 并点击设置

4. 修改配置项（如添加新的包裹命令、设置快捷键等）
   ↓

   ↓

```
1. 开启 Logseq 开发者模式
   ↓
2. 下载/克隆插件代码到本地
   ↓

### 路径四：WebURL 测试
```
1. 将插件托管到 Web 服务器
   ↓
2. 获取插件的 URL（指向 dist/index.html）
   ↓
3. 在 Logseq 中选择"从 URL 加载插件"

## 常见问题

### Q: 工具栏没有显示？
A: 请检查插件配置中的 `toolbar` 选项是否设置为 `true`。

### Q: 如何自定义快捷键？
A: 在配置中找到相应的命令，在 `binding` 字段中设置您想要的快捷键组合。

### Q: 配置修改后不生效？
A: 尝试重新加载插件或重启 Logseq。

### Q: 如何卸载插件？
A: 在 Logseq 插件市场中找到该插件，点击卸载即可。
